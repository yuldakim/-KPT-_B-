# KPT - Best Problem Practice
**Context: Cursor를 이용하여 하드 코딩에서 Vibe 코딩을 실습**

---

## Keep

### AI 코드 지원
- 코드 탐색: 직접 코드를 한 줄씩 읽지 않아도 AI가 전체 구조와 흐름을 빠르게 파악하고 설명해 주어, 낯선 코드베이스에 대한 진입 장벽이 낮아짐
- 리팩토링 제안: 작성한 코드의 개선 방향을 AI가 구체적으로 제안해 주어, 하드코딩 시 놓치기 쉬운 구조적 문제를 발견하고 개선하는 데 도움이 됨
- 코드 분석 및 에러 원인 파악 자동화: 에러 메시지와 스택 트레이스를 AI가 해석해 원인과 수정 방향을 제시해 주므로, 디버깅에 소요되는 시간과 심리적 부담이 크게 줄어듦

### 자동화
- 테스트 자동화: 테스트 케이스를 직접 작성하지 않아도 AI가 코드 흐름을 분석하여 자동으로 생성해 주므로, 테스트 작성 경험이 부족한 경우에도 품질 검증이 가능해짐
- 테스트 및 보고서 생성 자동화: 테스트 실행 결과를 정리된 보고서 형태로 자동 생성해 주어, 결과를 문서화하는 반복 작업에서 벗어날 수 있음
- 문서화/보고서: 코드에 대한 설명, 함수 주석, README 등 문서화 작업을 AI가 대신 작성해 주어, 하드코딩 시 미루기 쉬웠던 문서화를 자연스럽게 병행할 수 있음
- 필요한 모듈 자동 설치: 어떤 라이브러리가 필요한지 직접 검색하지 않아도 AI가 판단하여 설치까지 처리해 주므로, 환경 세팅 과정의 시행착오가 줄어듦

### 프롬프팅 역량
- PCTF / 7Step 등 다양한 프롬프팅 원칙을 학습하고 실제 사례를 경험함으로써, 처음에는 막연하게 느껴졌던 AI와의 대화 방식에 대한 이해가 생김. 습득한 원칙과 사례를 바탕으로 이후 자신의 프로젝트 성격에 맞게 프롬프트를 응용하고 발전시킬 수 있는 기반이 마련됨

## Problem

### 속도 및 자동화 한계
- Cursor AI의 처리 속도가 전체 프로세스 시간을 잡아먹음: 하드코딩에서는 내가 타이핑하는 속도가 곧 작업 속도였지만, Vibe 코딩에서는 AI가 코드를 생성하고 처리하는 대기 시간이 생각보다 길어 전체 작업 흐름이 끊기는 경우가 많았음
- 완전한 자동화가 되지 않아 사용자가 계속 지켜봐야 함: AI가 매 단계마다 사용자의 승인을 요구하는 구조이기 때문에, 자동화를 기대하고 시작했으나 실제로는 자리를 비울 수 없고 화면을 계속 주시해야 하는 상황이 반복됨

### 컨텍스트 관리
- Context 한계로 인한 대화창 단절: 문서나 코드가 길어질수록 AI가 이전 내용을 기억하지 못하는 한계에 부딪혀 새 대화창에서 다시 시작해야 하는 상황이 발생함. 하드코딩에서는 경험하지 못했던 새로운 제약 조건임
- 연속 작업의 어려움: 새 대화창에서 작업을 이어가려면 이전 맥락을 다시 설명해야 하는데, 초반에는 어디까지 전달해야 하는지 감을 잡기 어려워 작업의 연속성이 떨어졌음

### 워크플로우 및 자원 관리
- 워크플로우 설계의 중요성: 하드코딩은 코드를 잘 짜는 것이 핵심이었다면, Vibe 코딩에서는 AI에게 무엇을 어떤 순서로 시킬지를 먼저 설계하는 것이 전체 품질을 좌우함을 체감함
- 효과적인 AI 자원 사용의 중요성: 무분별하게 AI에게 요청을 쏟아내면 사용량 제한에 빠르게 도달하거나 처리 속도가 저하되는 문제가 발생하므로, AI를 자원으로 인식하고 효율적으로 활용하는 관점이 필요함

## Try

### 도구 다양화
- VSCode 기반 다른 AI CLI 사용 사례 연습: Cursor AI 하나에 의존하지 않고 VSCode 환경에서 사용할 수 있는 다양한 AI 도구들을 경험해 봄으로써, 작업 성격에 따라 적합한 도구를 선택할 수 있는 안목을 키움
- OpenRouter 등 오픈 모델 연계: 모든 작업을 하나의 메인 AI 모델에 맡기는 대신, OpenRouter 같은 플랫폼을 통해 단위 작업에 특화된 경량 모델을 조합하여 메인 모델의 부하와 사용량 제한 문제를 분산시킴
- 매크로 / Skill / Agent 활용 자동화: 반복적으로 수행하는 작업 패턴을 매크로나 Agent로 정의해 두면, 매번 승인을 기다리지 않고도 일정 범위의 작업을 자동으로 처리할 수 있는 환경을 구축할 수 있음

### 프로젝트 관리
- 요구사항 파악 우선: 코드를 짜기 전에 무엇을 만들어야 하는지를 명확히 정의하는 것이 Vibe 코딩에서는 더욱 중요함. AI에게 모호한 요구사항을 전달하면 원하지 않는 방향으로 코드가 생성되는 경우가 많았기 때문에, 요구사항 정의에 충분한 시간을 투자함
- 프로세스 구축: 요구사항이 정해지면 AI에게 한 번에 모든 것을 맡기지 않고, 작업을 단계별로 나누어 순서와 의존관계를 먼저 설계함
- TODO 리스트 작성 및 단계별 점검: 전체 작업을 TODO 리스트로 관리하고 각 단계 완료 시점에 결과를 확인하는 습관을 들여, 중간에 방향이 틀어지는 것을 조기에 발견할 수 있도록 함
- 문서 작성 템플릿 준비: 반복적으로 생성해야 하는 문서 유형에 대해 템플릿을 미리 준비해 두면, AI가 일관된 형식으로 문서를 생성하도록 유도할 수 있음
