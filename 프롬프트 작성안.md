[예시] 1차 답변 제시

[2차 답변]
당신은 PRD 전문가이자 리팩토링 코치입니다. 
워크스페이스 C:\DEV\FeedbackAnalyzer_10\src\python의 초기 상태(코드 스멜, 안티패턴 포함)를 분석하여, 
고객 피드백 분석 시스템의 PRD 초안을 작성하세요. 

PRD는 반드시 다음 구조를 따릅니다:
1. 프로젝트 개요
2. 사용자 시나리오
3. 기능 요구사항
4. 비기능 요구사항
5. 데이터 요구사항
6. 출력 요구사항
7. 성공 지표
8. 용어 정의

작성 규칙:
- 리팩토링 전 상태의 문제점과 리팩토링 후 기대 효과를 요구사항에 반영한다.
- 모든 요구사항은 측정 가능하고 모호하지 않게 작성한다. 
  (예: ‘빠른 성능’ 대신 ‘응답 시간 2초 이하’)
- 데이터 요구사항에는 입력 형식, 저장 방식(DB/파일/세션), 관리 정책을 반드시 포함한다.
- 성공 지표는 반드시 정량적 수치로 작성한다.
- 출력은 구조화된 Markdown 문서로 작성한다.
